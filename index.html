Roadmap

Nächste sinnvolle Schritte zur Bereinigung, Dokumentation und Stabilisierung deines Homelabs.

Phase 1: Kritische Basis bereinigen Fast fertig
  • IP-Konflikt zwischen nodjs-dev und mariadb02 behoben
  • Neue IP von nodjs-dev dokumentiert
  • MariaDB-Nodes geprüft
  • Backup-Jobs für kritische Container geprüft
Phase 2: Container bereinigen Next
  • de prüfen und sprechend umbenennen
  • Gestoppte Container prüfen: web01, nt-monitoring, minio
  • Weitere Container prüfen: apps01, test-db, proxy-wi-dev
  • Entscheiden: behalten, löschen, dokumentieren oder umbenennen
Phase 3: NetBox abgleichen IPAM
  • NetBox mit aktuellem IP-Plan vergleichen
  • Fehlende IPs ergänzen
  • Veraltete Einträge markieren oder entfernen
  • Container, VMs und Dienste korrekt zuordnen
Phase 4: Monitoring dokumentieren Monitoring
  • Kritische Dienste definieren
  • Monitoring-Check je Dienst dokumentieren
  • Check-Typ eintragen: Ping, HTTP, TCP, DNS, SNMP oder DB
  • Alarmierung dokumentieren: Telegram, E-Mail oder Discord
Phase 5: Doku aufräumen Docs
  • Docker-Bridge-IPs aus der Hauptdoku entfernen
  • Dienste mit URLs ergänzen
  • Einheitliches Format für Dienst-Dokumentation verwenden
  • Backup-Aufbewahrung je Dienst ist bereits dokumentiert
Phase 6: Diagramme ergänzen Visualisierung
  • Netzwerkdiagramm ergänzen
  • Mermaid-Diagramm im Repo erstellen
  • Optional Screenshots wichtiger Dienste hinzufügen
  • Service-Übersicht visuell verständlicher machen
